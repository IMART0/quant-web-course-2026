Концепция проекта: Сайт школы «Квант»

Мы разрабатываем полноценный интерфейс для летней школы. Сегодня мы создадим статический макет, который позже интегрируем с Django:

  • Главная страница: Общая информация, направления обучения (физ-мат, IT, хим-био).

Наша цель сегодня: Научиться правильно размечать текст, создавать семантический каркас сайта и проектировать интерактивные формы, которые подготовят нас к приему POST-запросов в Django.

Что такое HTML5? Роль в веб-приложении

HTML (HyperText Markup Language) — это язык разметки, определяющий структуру и каркас веб-страницы. Он интерпретируется браузером.

  • Это не язык программирования! В нем нет переменных, циклов или логических условий.
  • Аналогия: HTML — это несущий каркас и стены дома; CSS — его отделка и дизайн; JavaScript — интерактивное оборудование; Django — фундамент и инженерные коммуникации.
Связь с Django: Django-шаблоны отдают именно HTML-код. Браузер пользователя видит только HTML, CSS и JS, но не видит код на Python!

Базовая структура HTML-документа

<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <title>Сайт школы Квант</title>
</head>
<body>
    <!-- Контент страницы пишется здесь -->
    <h1>Добро пожаловать в Квант!</h1>
</body>
</html>
  • <!DOCTYPE html> — объявление стандарта HTML5.
  • <html> — корневой элемент.
  • <head> — метаданные (кодировка, заголовок вкладки, стили).
  • <body> — видимый контент.

Анатомия тегов и атрибуты

Теги бывают парными (контейнеры для текста/других тегов) и одиночными (вставляют элементы в страницу).

<имя_тега имя_атрибута="значение"> Содержимое </имя_тега>

Примеры для сайта школы «Квант»:

  • Парный тег (ссылка на расписание):
    <a href="/schedule.html" target="_blank">Посмотреть расписание</a>
  • Одиночный тег (логотип лагеря):
    <img src="logo.png" alt="Логотип Кванта" width="150">

Вложенность тегов и дерево DOM

Правило «матрешки»: теги, которые открываются позже, должны закрываться раньше.

❌ Неверно: <p>Мы изучаем <strong>HTML5</p></strong>
✅ Верно: <p>Мы изучаем <strong>HTML5</strong></p>

Браузер строит из тегов DOM (Document Object Model) — древовидную структуру, где элементы являются родителями или потомками друг друга.

Типы элементов:

  • Блочные (block): Занимают всю ширину строки, начинаются с новой строки (<div>, <p>, <h1>, <ul>).
  • Строчные (inline): Располагаются внутри текста, ширина зависит от контента (<span>, <a>, <strong>, <em>).

Текстовая разметка на сайте «Кванта»

  • Заголовки: <h1> до <h6>. <h1> — главный заголовок (название страницы). Он должен быть строго один!
  • Абзацы: Тег <p> для группировки предложений.
  • Списки:
    • <ul> — маркированный список направлений.
    • <ol> — нумерованный список шагов регистрации.
    • <li> — элемент списка (всегда внутри <ul>/<ol>).
<h2>Наши направления</h2>
<ul>
    <li>Физико-математическое</li>
    <li>Информационные технологии</li>
    <li>Химико-биологическое</li>
</ul>

Ссылки и изображения

Изображения (<img>):

  • Атрибут src — путь к файлу.
  • Атрибут alt — альтернативный текст (для SEO, скринридеров и при сбоях сети). Обязателен!

Ссылки (<a>):

  • Атрибут href указывает на URL.
  • Атрибут target="_blank" открывает ссылку в новой вкладке.
Пример связывания страниц: 
<a href="event.html">
  <img src="camp.jpg" alt="Кампус лагеря Квант">
  <p>Узнать подробнее о смене</p>
</a>

Проблема «div-супа» и семантика HTML5

Раньше разработчики создавали всю структуру с помощью безликих тегов <div>:

<div class="header">...</div>
<div class="menu">...</div>
<div class="sidebar">...</div>

Это делало разметку «слепой» для поисковиков и скринридеров.

Почему это плохо? Поисковые роботы (Яндекс, Google) не могут понять, где важный контент, а где реклама или меню. Программы для незрячих пользователей не могут озвучить структуру страницы.

Семантические элементы HTML5

HTML5 вводит теги, которые сообщают браузеру и поисковикам о роли содержимого:

  • <header> — шапка сайта (логотип, название лагеря).
  • <nav> — навигационная панель (меню страниц).
  • <main> — уникальное основное содержимое страницы.
  • <aside> — боковая панель (список кружков, контакты).
  • <footer> — подвал (копирайт, ссылки на соцсети).
Принцип семантики: Использовать тег, соответствующий смыслу данных. Это улучшает SEO (поисковую выдачу) и доступность сайта.

Разметка контента: Section и Article

Внутри тега <main> мы структурируем данные с помощью:

  • <section> — логический блок или раздел (например, секция «О лагере», секция «Контакты»). Обычно содержит свой заголовок <h2>-<h6>.
  • <article> — независимый, самодостаточный блок контента (новость, анонс лекции, пост в блоге). Его можно вырезать и перенести на другой сайт, и смысл не потеряется.
<section id="news">
    <h2>Новости лагеря</h2>
    <article>
        <h3>Открытие IT-смены!</h3>
        <p>Сегодня стартовала смена...</p>
    </article>
</section>

Анатомия веб-форм: Тег <form>

Форма (<form>) — это область интерфейса для ввода и передачи данных на сервер.

  • Атрибут action — URL-адрес обработчика на сервере (например, Django view).
  • Атрибут method — метод HTTP-запроса:
    • GET: Данные передаются в строке URL (поиск, фильтрация).
    • POST: Данные передаются скрыто в теле запроса (регистрация, отправка паролей).
Форма записи в Квант: 
<form action="/register/" method="POST">
    <!-- Поля ввода будут здесь -->
    <button type="submit">Отправить</button>
</form>

Текстовые поля ввода

Элемент <input> — основной интерактивный элемент. Тип ввода определяется атрибутом type.

  • Обычный текст (Имя):
    <input type="text" placeholder="Введите имя">
  • Пароль (скрытые символы):
    <input type="password" placeholder="Придумайте пароль">
  • Электронная почта (с автовалидацией):
    <input type="email" placeholder="example@mail.com">
Элемент <label>: связывает текстовое описание с полем с помощью атрибута for и id. Это увеличивает область клика! 
<label for="username">Логин:</label>
<input id="username" type="text">

Специализированные типы ввода

HTML5 предлагает множество встроенных типов ввода, которые автоматически проверяются браузером и предоставляют удобные виджеты (например, календарь на смартфонах):

  • Числа (type="number"): Ввод возраста или класса школьника. Поддерживает атрибуты min и max.
    <input type="number" min="7" max="11" value="9">
  • Дата (type="date"): Выбор даты рождения или начала смены с помощью встроенного календаря.
    <input type="date" min="2026-06-01" max="2026-08-31">
  • Телефон (type="tel"): Поле для номера телефона.
    <input type="tel" pattern="[0-9]{11}" placeholder="89991234567">

Чекбоксы и радиокнопки

Радиокнопки (type="radio"): Выбор одного варианта из нескольких. Важно: у них должен быть одинаковый атрибут name для объединения в группу!

<p>Выберите направление:</p>
<input type="radio" id="dir-it" name="direction" value="it">
<label for="dir-it">IT-технологии</label>
<input type="radio" id="dir-ph" name="direction" value="physics">
<label for="dir-ph">Физика</label>

Чекбоксы (type="checkbox"): Независимый выбор нескольких вариантов (или согласие с правилами).

<input type="checkbox" id="rules" name="agree" required>
<label for="rules">Согласен с правилами</label>

Выпадающие списки и многострочный ввод

Выпадающий список (<select> и <option>):
Удобен для экономии места на экране, когда вариантов много.

<label for="camp-shift">Смена:</label>
<select id="camp-shift" name="shift">
    <option value="1">Июньская научная</option>
    <option value="2">Июльская IT-смена</option>
    <option value="3">Августовская хим-био</option>
</select>

Текстовое поле (<textarea>):
Используется для длинного произвольного текста (мотивационное письмо, вопросы, комментарии).

<label for="letter">О себе:</label>
<textarea id="letter" name="bio" rows="4" cols="30" placeholder="Расскажите о своих достижениях..."></textarea>

Валидация данных в HTML5

Валидация — это проверка корректности введенных данных до их отправки на сервер. HTML5 поддерживает встроенные атрибуты проверки:

  • required — поле не может быть отправлено пустым.
  • minlength / maxlength — ограничение по числу символов (например, пароль не менее 8 символов).
  • min / max — числовые ограничения.
  • pattern — проверка с помощью регулярных выражений (например, формат телефона).
Важное предупреждение: Клиентская валидация легко обходится продвинутыми пользователями (через DevTools). Поэтому бэкенд на Django всегда должен дублировать проверки!

Связь с Django: Ключевая роль атрибута name

Как Django узнает, какие данные прислал пользователь?

Каждое поле формы обязано иметь атрибут name. При отправке формы браузер формирует пары ключ=значение:

имя_поля_1=значение_1&имя_поля_2=значение_2

В Python-коде Django мы извлекаем эти данные по ключам:

student_name = request.POST.get('username')
Правило разработчика:
  • id — используется для связывания с <label> и стилей (CSS).
  • name — передается на сервер и используется бэкендом (Django).
Без name поле вообще не будет отправлено на сервер!

Реализация формы регистрации

<form action="/apply/" method="POST">
  <p>
    <label for="fio">ФИО:</label>
    <input id="fio" type="text" name="student_fio" required>
  </p>
  <p>
    <label for="email">Email:</label>
    <input id="email" type="email" name="student_email" required>
  </p>
  <p>
    <label for="shift">Смена:</label>
    <select id="shift" name="selected_shift">
      <option value="it">IT-смена</option>
      <option value="fm">Физ-мат смена</option>
    </select>
  </p>
  <button type="submit">Зарегистрироваться</button>
</form>

Почему именно так?

  • Теги сгруппированы через абзацы <p> для базового отступа.
  • Все поля валидируются (required, type="email").
  • Каждое поле имеет уникальный name, который совпадает с именами полей в бэкенде.
