Зачем нам CSS3 на сайте «Кванта»?

CSS3 (Cascading Style Sheets) превращает простую HTML-разметку нашего лагерного сайта в современный адаптивный веб-интерфейс.

  • Отделяет структуру ("О лагере", "Расписание", "Форма") от внешнего вида.
  • Выстраивает элементы: хедер, карточки преподавателей в ряд, расписание в сетку.
  • Адаптирует сайт под экраны телефонов, планшетов и ноутбуков.

Аналогия проекта

HTML — это бетонный каркас учебного корпуса «Кванта» и голые стены. CSS — это фасад, покраска, окна, расположение комнат, расстановка парт и указатели в коридорах.

Находим элементы: Базовые селекторы

Чтобы стилизовать наш сайт, нужно точно указать браузеру, какие HTML-элементы мы оформляем:

  • По тегу: стилизует структуру. 
    header { background: #2c3e50; }
section { padding: 40px; }
  • По классу (.): многократные элементы (карточки учителей). 
    .teacher-card { border: 1px solid; }
.form-input { padding: 10px; }
  • По ID (#): уникальный элемент на странице. 
    #registration-section {
  background-color: #f9f9f9;
}
  • Группировка (,): общие стили заголовков. 
    h1, h2, h3 {
  font-family: 'Segoe UI';
}

Связи в разметке: Комбинаторы

Стилизуем элементы в зависимости от их взаимного расположения в структуре сайта:

  • Потомок (пробел): стилизует все ссылки внутри навигации (на любой глубине). 
    .nav-menu a { color: #3498db; }
  • Дочерний селектор (>): стилизует только прямых детей. 
    .teacher-card > h3 { margin-top: 15px; } /* Только h3 внутри карточки */
  • Соседний селектор (+): стилизует первый элемент сразу после заголовка. 
    h2 + p { color: #7f8c8d; } /* Вводный абзац сразу под h2 */

Интерактив: Псевдоклассы и Псевдоэлементы

Псевдоклассы (состояние элемента)

  • a:hover — наведение мыши на пункт меню.
  • input:focus — подсветка рамки при вводе имени/email.
  • button:active — эффект нажатия на кнопку регистрации.

Псевдоэлементы (виртуальный контент)

  • ::before и ::after вставляют декоративные элементы или текст. 
    /* Звездочка для обязательных полей */
.required::after {
  content: " *";
  color: #e74c3c;
}

Блочная модель (Box Model) карточки

Каждый элемент сайта «Кванта» (например, карточка преподавателя) — это прямоугольник из 4 слоев:

  • Content (Содержимое): фото и имя (width, height).
  • Padding (Внутренний отступ): воздух от текста до границы карточки.
  • Border (Рамка): линия контура карточки.
  • Margin (Внешний отступ): расстояние между соседними карточками.
Margin (Внешний отступ)
Border (Граница)
Padding (Внутренний отступ)
Content (Фото + Имя)

box-sizing: border-box на практике

content-box (по умолчанию)

Размеры рамки и внутренние отступы прибавляются снаружи к ширине.

/* Реальная ширина на экране:
   300px + 20px*2 + 2px*2 = 344px! 
   Форма вылезет за пределы экрана! */
.form-input {
  width: 300px;
  padding: 20px;
  border: 2px solid;
}

border-box (стандарт верстки)

Ширина элемента фиксируется. Отступы и границы уходят внутрь.

/* Реальная ширина на экране:
   ровно 300px! 
   Идеально ложится в сетку. */
.form-input {
  box-sizing: border-box;
  width: 300px;
  padding: 20px;
  border: 2px solid;
}
Всегда сбрасываем модель в начале CSS: * { box-sizing: border-box; }

Flexbox: Управляем осями и строками

CSS Flexible Box Layout (Flexbox) — одномерная раскладка. Позволяет выстроить элементы в ряд или столбец.

  • Одномерный: работает строго по одной оси в один момент времени.
  • Главная ось (Main Axis): направление движения элементов.
  • Поперечная ось (Cross Axis): перпендикулярна главной.

Как включить?

Задаем родителю (например, хедеру сайта):

.site-header {
  display: flex;
}

Направление и перенос во Flexbox

flex-direction (Направление)

  • row (дефолт) — элементы стоят горизонтально (логотип и меню).
  • column — элементы стоят вертикально (версия для телефонов).
.site-header {
  flex-direction: column;
}

flex-wrap (Перенос строк)

  • nowrap (дефолт) — сжимать элементы до упора в один ряд.
  • wrap — переносить карточки преподавателей на новую строку, если они не помещаются.
.teacher-list {
  display: flex;
  flex-wrap: wrap;
}

Выравнивание по главной оси: justify-content

Распределяем свободное пространство между логотипом сайта и меню навигации:

  • flex-start — все элементы прижаты к левому краю.
  • flex-end — прижаты к правому краю.
  • center — собраны в центре.
  • space-between — крайние элементы прижаты к границам (логотип слева, меню справа).
  • space-around / space-evenly — равные отступы вокруг элементов.
.site-header {
  display: flex;
  justify-content: space-between; /* Логотип и меню разъедутся по краям */
}

Выравнивание по поперечной оси

align-items (для всех элементов)

Выравнивает логотип и ссылки хедера по вертикали:

  • stretch (дефолт) — растянуть по высоте.
  • center — отцентровать по высоте хедера (самое популярное).
  • flex-start / flex-end — прижать к верху/низу.

align-self (индивидуально)

Переопределяет выравнивание для одной конкретной кнопки или иконки:

.login-button {
  align-self: flex-end; /* прижать вниз */
}

Управление гибкостью элементов

Задаем правила поведения для самих карточек внутри flex-контейнера:

  • flex-grow: коэффициент роста. Насколько элемент может вырасти, чтобы занять пустое место (по умолчанию 0).
  • flex-shrink: способность сжиматься при нехватке места (по умолчанию 1).
  • flex-basis: базовый размер элемента до распределения пространства (например, 300px).

Shorthand flex

/* grow | shrink | basis */
.teacher-card {
  flex: 1 1 300px;
}

Карточка пытается занять 300px, но может расти и сжиматься при необходимости.

CSS Grid Layout: Двумерные сетки

CSS Grid Layout управляет сеткой из колонок и строк одновременно. Идеален для сетки Расписания смен и мероприятий.

Flexbox vs Grid

  • Flexbox: одномерный (верстаем Хедер или Карточки).
  • Grid: двумерный (верстаем общую сетку страницы, календари и расписания).

Включаем сетку:

.schedule-grid {
  display: grid;
}

Размечаем сетку: repeat() и fr

Фракции (fr) и Шаблоны

  • grid-template-columns — ширина колонок.
  • fr (fraction) — доля свободного места. 
    /* 3 колонки: 1-я 200px,
   остальные делят остаток 1:2 */
grid-template-columns: 200px 1fr 2fr;

Полезные функции

  • repeat(кол-во, размер) — автоповтор колонок.
  • minmax(min, max) — гибкий адаптивный размер.
/* 3 одинаковые колонки для расписания */
grid-template-columns: repeat(3, 1fr);

Управление ячейками и отступами (gap)

Отступы сетки (gap)

Задает расстояние между строками и колонками без margin:

.schedule-grid {
  display: grid;
  gap: 15px; /* отступы 15px */
}

Объединение ячеек

Растягиваем важное мероприятие в расписании на две колонки или строки:

.main-lecture {
  grid-column: span 2; /* Занять 2 колонки */
}
.full-day-event {
  grid-column: 1 / 4; /* От 1-й линии до 4-й */
}

Адаптивность: Подготовка к мобильным

Наш сайт «Кванта» должен удобно читаться на телефонах ребят и планшетах преподавателей.

Для корректного масштабирования на мобильных устройствах всегда пишем этот тег в <head>:

<meta name="viewport" content="width=device-width, initial-scale=1.0">

Если забыть этот тег:

Мобильный телефон симулирует огромный экран компьютера (~980px) и сожмет весь сайт до микроскопического размера. Читать расписание или заполнить форму будет невозможно.

Медиа-запросы и брейкпоинты

Позволяют применять правила CSS выборочно, в зависимости от ширины экрана устройства:

/* Базовый стиль под мобильные */
.teacher-list {
  flex-direction: column;
}

/* От планшетов и шире (>= 768px) */
@media screen and (min-width: 768px) {
  .teacher-list {
    flex-direction: row;
  }
}

Mobile First подход

Сначала пишем простые стили в одну колонку (для телефонов). Затем с помощью медиа-запросов min-width расширяем интерфейс до многоколоночного для компьютеров.

Адаптивный макет Сайта Кванта

Форма регистрации (Mobile vs Desktop)

/* Базовый CSS (Телефоны) */
.reg-form {
  display: grid;
  grid-template-columns: 1fr; /* в 1 колонку */
}

@media screen and (min-width: 768px) {
  .reg-form {
    grid-template-columns: 1fr 1fr; /* в 2 колонки */
  }
  .submit-btn {
    grid-column: span 2; /* кнопка на всю ширину */
  }
}

Хедер сайта (Mobile vs Desktop)

/* Базовый CSS (Телефоны) */
.site-header {
  display: flex;
  flex-direction: column;
  align-items: center;
}

@media screen and (min-width: 600px) {
  .site-header {
    flex-direction: row;
    justify-content: space-between;
  }
}

Резюме и переходим к практике!

  • Селекторы: находят элементы на нашем Квант-сайте.
  • Box Model: отступы и границы. Помним про border-box.
  • Flexbox: идеален для хедера и горизонтальных блоков.
  • Grid Layout: лучший выбор для сетки расписания и форм.
  • @media: делает элементы гибкими на любых устройствах.
  • Mobile First: начинаем верстку с мобильных версий.
