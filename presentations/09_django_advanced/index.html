Что такое Django Template Language (DTL)?

DTL — это встроенный в Django шаблонизатор, предназначенный для динамической генерации HTML-страниц на основе данных из View.

Философия DTL:

  • Разделение логики: Шаблон отвечает за представление, Python-код — за логику.
  • Ограниченность: В DTL намеренно нельзя писать произвольный Python-код (например, вызывать функции с аргументами или делать сложные вычисления).
  • Безопасность: Автоматическое экранирование HTML для защиты от XSS-атак.

Почему это важно?

Если в шаблоне разрешить писать чистый Python (как в некоторых других веб-фреймворках), разметка быстро превратится в нечитаемый спагетти-код, а дизайнеры не смогут работать с HTML-файлами.

Переменные в DTL

Переменные выводятся с помощью двойных фигурных скобок: {{ переменная }}. Django подставляет значение, переданное в контексте.

Универсальный синтаксис точки (.)

Django использует точку для доступа к элементам любой структуры данных:

  • Ключ словаря: {{ user.name }} → user['name']
  • Атрибут объекта: {{ user.email }} → user.email
  • Элемент списка по индексу: {{ users.0 }} → users[0]
  • Метод объекта (без скобок): {{ user.get_full_name }} → user.get_full_name()

Важная деталь

Методы вызываются без скобок (). Если метод требует аргументов, его нельзя вызвать напрямую в шаблоне! Для этого используются фильтры или кастомные теги.

Теги в DTL (Синтаксис {% tag %})

Теги управляют логикой рендеринга шаблона. Они записываются внутри фигурных скобок со знаком процента: {% ... %}.

Основные категории тегов:

  • Логические ветвления: {% if %}, {% elif %}, {% else %}
  • Циклы: {% for %}, {% empty %}
  • Управление структурой: {% block %}, {% extends %}, {% include %}
  • Работа со статикой и URL: {% load static %}, {% url %}

Закрытие тегов

Многие теги в DTL являются парными и требуют явного закрытия. Например, для {% if %} нужен {% endif %}, а для {% for %} — {% endfor %}. Незакрытый тег вызовет синтаксическую ошибку шаблонизатора (TemplateSyntaxError).

Логические ветвления: {% if %}

{% if user.is_authenticated %}
  <p>Привет, {{ user.username }}!</p>
  <a href="/logout/">Выйти</a>
{% elif user.is_banned %}
  <p>Ваш аккаунт временно заблокирован.</p>
{% else %}
  <p>Войдите на сайт или зарегистрируйтесь.</p>
  <a href="/login/">Войти</a>
{% endif %}

Логические операторы:

  • and, or, not — стандартная булева логика.
  • in и not in — проверка вхождения в последовательность.
  • Операторы сравнения: ==, !=, <, >, <=, >=.
  • Можно группировать выражения с помощью круглых скобок.

Циклы: {% for %}

Используется для перебора списков, кортежей или словарей в шаблоне.

Перебор списка объектов:

<ul>
{% for book in books %}
  <li>{{ book.title }} ({{ book.author }})</li>
{% endfor %}
</ul>

Перебор словаря (ключ, значение):

<dl>
{% for key, val in settings.items %}
  <dt>{{ key }}</dt>
  <dd>{{ val }}</dd>
{% endfor %}
</dl>

Счетчики внутри цикла: forloop

Внутри любого цикла {% for %} доступна специальная переменная forloop, содержащая полезную информацию о текущем шаге:

  • forloop.counter — текущая итерация (индекс начинается с 1).
  • forloop.counter0 — текущая итерация (индекс начинается с 0).
  • forloop.first — True, если это первый элемент цикла.
  • forloop.last — True, если это последний элемент цикла.
  • forloop.parentloop — доступ к счетчику внешнего цикла (при вложенности).
{% for item in items %}
  <div class="item {% if forloop.first %}highlight{% endif %}">
    Шаг №{{ forloop.counter }}: {{ item }}
  </div>
{% endfor %}

Конструкция {% empty %} в цикле

Частая задача: вывести сообщение, если список пуст. Вместо того, чтобы писать {% if items %} {% for ... %} {% endif %}, в DTL есть элегантное решение.

Блок {% empty %} выполняется тогда, когда список пуст или равен None.

<h3>Список участников:</h3>
<ul>
{% for user in active_users %}
  <li>{{ user.username }}</li>
{% empty %}
  <li>К сожалению, активных участников нет.</li>
{% endfor %}
</ul>

Фильтры в DTL

Фильтры используются для преобразования или форматирования значения переменной перед её выводом на экран.

Синтаксис: символ вертикальной черты (pipe) | после переменной.

{{ variable|filter_name }}

Некоторые фильтры принимают аргументы через двоеточие:

{{ variable|filter_name:"argument" }}

Цепочки фильтров

Фильтры можно применять последовательно один за другим:

{{ name|lower|capfirst }}

Сначала строка приводится к нижнему регистру, а затем первая буква делается заглавной.

Популярные встроенные фильтры

  • default — значение по умолчанию, если переменная пустая: 
    {{ bio|default:"Информации нет" }}
  • length — возвращает длину списка или строки: 
    Всего: {{ users|length }}
  • truncatewords — обрезает строку до N слов: 
    {{ article|truncatewords:20 }}
  • date — форматирует дату и время: 
    {{ pub_date|date:"d.m.Y H:i" }}
  • slice — срез списка или строки: 
    {{ news|slice:":3" }}
  • lower / upper — меняет регистр букв: 
    {{ title|upper }}

Безопасный вывод и XSS-защита

По умолчанию Django экранирует все специальные HTML-символы во избежание XSS (Cross-Site Scripting):

Если text = "<b>Привет</b>", то {{ text }} выведет на экран сырой текст с тегами: &lt;b&gt;Привет&lt;/b&gt;.

Как сказать Django отрендерить теги?

  • Фильтр safe: {{ text|safe }}
  • Тег autoescape: 
    {% autoescape off %}
  {{ text }}
{% endautoescape %}

Внимание! Безопасность!

Никогда не применяйте фильтр safe к данным, которые ввёл пользователь (например, комментарии или имена). Иначе злоумышленник сможет вставить вредоносный JavaScript-код (<script>...</script>) и украсть куки посетителей!

Наследование шаблонов: DRY-принцип

Проблема дублирования кода:

Обычно у всех страниц сайта одинаковая шапка (header), подвал (footer) и боковая панель (sidebar). Дублировать этот HTML-код на каждой странице — грубое нарушение принципа DRY (Don't Repeat Yourself).

При любом изменении дизайна (например, ссылки в меню) придётся вручную переписывать десятки файлов.

Решение в Django

Создать единый родительский шаблон (каркас сайта), а остальные страницы будут наследоваться от него, переопределяя только изменяющиеся части (блоки контента).

Шаблон-родитель: base.html

Родительский шаблон описывает общую структуру страницы. В местах, где контент страниц будет различаться, расставляются теги {% block <имя_блока> %} ... {% endblock %}.

<!DOCTYPE html>
<html>
<head>
  <title>{% block title %}Мой сайт{% endblock %}</title>
</head>
<body>
  <header>Шапка сайта с логотипом и меню</header>
  
  <main class="content">
    {% block content %}
      <p>Здесь будет основной контент страницы.</p>
    {% endblock %}
  </main>
  
  <footer>© 2026 ЛФМШ "Квант"</footer>
</body>
</html>

Шаблон-потомок: Тег {% extends %}

Чтобы создать страницу на базе base.html, мы используем тег {% extends "путь/к/base.html" %} в самой первой строчке файла.

{% extends "base.html" %}

{% block title %}Главная страница — Мой сайт{% endblock %}

{% block content %}
  <h1>Добро пожаловать на наш сайт!</h1>
  <p>Это контент главной страницы. Он автоматически встанет внутрь каркаса base.html.</p>
{% endblock %}

Правило: Тег {% extends %} должен быть самым первым тегом в шаблоне. Всё, что написано вне блоков {% block %} в файле-потомке, Django просто проигнорирует.

Расширение контента родителя: {{ block.super }}

По умолчанию блок в потомке полностью стирает и заменяет контент родительского блока.

Если мы хотим добавить новый контент к родительскому (а не стереть его), используется специальная переменная {{ block.super }}.

Пример (подключение стилей):

{% block styles %}
  
  {{ block.super }}
  
  <link rel="stylesheet" href="page.css">
{% endblock %}

Включение шаблонов: Тег {% include %}

Тег {% include %} позволяет загрузить и отрендерить другой шаблон внутри текущего. Это полезно для вынесения повторяющихся мелких компонентов интерфейса (виджетов).

Примеры компонентов: карточка товара, форма подписки, боковая панель, блок комментариев.

Пример использования:

<div class="catalog">
  {% for product in products %}
    {% include "includes/product_card.html" with item=product %}
  {% endfor %}
</div>

С помощью конструкции with можно передать в подключаемый шаблон дополнительные переменные.

Что такое статические файлы (Static Files)?

Статические файлы — это файлы, которые передаются клиенту "как есть" (без обработки сервером шаблонов). К ним относятся:

  • CSS стили
  • JavaScript файлы
  • Изображения (PNG, JPG, SVG, WebP)
  • Шрифты (WOFF, TTF)

Статика vs Медиа

Статика создаётся разработчиком и является частью приложения (логотип сайта, скрипты логики, стили кнопок).

Медиа (Media) загружается пользователями сайта в процессе работы (аватарки пользователей, картинки к товарам).

Настройка статики в settings.py

Для работы со статикой в файле настроек проекта необходимо объявить три ключевые переменные:

# settings.py

# URL-префикс для статических файлов в браузере (например: /static/css/main.css)
STATIC_URL = 'static/'

# Списки директорий на диске, где Django ищет файлы во время разработки
STATICFILES_DIRS = [
    BASE_DIR / 'static',
]

# Папка, куда соберутся все файлы для продакшена при вызове команды collectstatic
STATIC_ROOT = BASE_DIR / 'staticfiles'

Использование тега {% static %}

Чтобы подключать статические файлы, нужно загрузить библиотеку тегов с помощью {% load static %} в начале шаблона.

{% load static %}
<!DOCTYPE html>
<html>
<head>
  <!-- Подключаем CSS стили -->
  <link rel="stylesheet" href="{% static 'css/styles.css' %}">
</head>
<body>
  <!-- Выводим изображение -->
  <img src="{% static 'images/logo.png' %}" alt="Логотип">
  
  <!-- Подключаем JS скрипт -->
  <script src="{% static 'js/main.js' %}"></script>
</body>
</html>

Резюме урока и практические задачи

  • DTL используется для рендеринга данных во View.
  • Синтаксис {{ ... }} выводит значения, а {% ... %} управляет логикой.
  • Циклы {% for %} имеют полезный объект forloop и блок {% empty %}.
  • Фильтры (например, safe, date, default) трансформируют вывод.
  • Наследование с {% extends %} и {% block %} исключает дублирование кода.
  • Статические файлы подключаются через {% load static %} и {% static '...' %}.
